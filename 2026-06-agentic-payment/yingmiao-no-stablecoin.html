Position paper

Agentic Payment 的最终赢家
不需要稳定币

卡组织已经在给 Agent 发 token（Visa Intelligent Commerce、Mastercard Agent Pay），Stripe 在给 Agent 发虚拟卡和 Shared Payment Token。如果这些就够了，稳定币解决的「快、终局、便宜跨境」在主流 agentic 场景里要么不是瓶颈，要么反而是负债。

核心命题：Agent 缺的是授权与责任，不是新货币轨道。前者是卡网络 60 年的主场，后者才是稳定币的主场——而 agentic payment 的胜负手在前者。
The argument

三条论据：稳定币解决的是一个不存在的瓶颈

稳定币的全部卖点 = 结算终局性 + 可编程性 + 低成本跨境。逐条证明：在「Agent 替人花钱」场景里，这三件事要么已被卡 token 覆盖，要么不是瓶颈。

1

Agent 需要的是「授权」，不是「新货币轨道」

把 agentic payment 拆开，难点从来不在「钱怎么转」，而在三个问题：这 Agent 真的是我授权的吗？它能花多少、买什么、何时停？出错谁负责、能不能退？这三件事稳定币一件都不解决。

而卡组织的方案恰好就是冲着这三件事去的——它们卖的不是「转账」，是「可信、可约束、可撤销的代理身份」：

  • Visa Intelligent Commerce：给每个 Agent 发绑定特定凭证的 token，「每个消费者设定限额，Visa 管理其余」——把授权约束写进网络层。
  • Mastercard Agent Pay：建立在 tokenization + trust + agentic commerce 三角上，是 AI-native 的可信代理基础设施。
  • Stripe Shared Payment Tokens（ACP）：让商户能安全接受 Agent 发起的交易，token 受范围限定。
→ 赢家比的是「谁的授权/信任层最完善」，这是卡网络的主场。
2

可逆性是 feature，不是 bug——「终局性」在这里是负债

稳定币最自豪的特性是不可逆的即时结算。在一个会犯错的 AI 自动花钱的场景里，这是致命缺陷。

想象 Agent 被 prompt injection 劫持、把「买一杯奶茶」理解成「买一箱」、或自动下错单。卡网络有一整套成熟的拒付（chargeback）、争议、退款机制兜底，消费者受 Reg E / Reg Z 等法律保护。稳定币里，钱一旦上链就拿不回来。

这是反直觉但极强的一击：Agent 越自主、越容易出错，可逆的卡网络价值就越高，稳定币的「终局性」反而像是把刀递给一个手抖的人。当你让 AI 替你花钱，你比任何时候都更需要可回滚的轨道，而不是更不需要。

→ 自主性越强 → 越需要可逆 → 卡网络越不可替代。
3

成本与跨境优势，被「够用就行 + 卡组织主动降本」消解

稳定币最硬的论据是微支付 + 跨境太贵（2.9% + $0.30 的固定费摧毁小额交易）。这是真实痛点，但要证明「赢家不需要稳定币」，只需证明主流场景里它不致命。

第一点（场景）：绝大多数 agentic commerce 是普通消费购物——AI 订机票、买日用品、续订阅，客单价 $10–$1000，卡费率完全可接受，且用户已经在用卡。用「未来可能很大的边缘场景（M2M 微支付）」去论证「主流赢家必须用稳定币」，是用尾部否定主体，逻辑不成立。

第二点（卡组织正在用 token 化主动压低成本曲线）——这是被低估的关键，下一节专门展开。

→ 赢家服务 99% 的主流交易，这部分卡网络够用且体验更好。
Argument 3 · deep dive

卡组织正在用 token 化「降本增效」

稳定币阵营默认「卡网络又贵又笨且不会变」。但 network tokenization 正在主动重构成本结构，且优化方向恰好命中 Agent 痛点。

先厘清一个致命混淆：卡组织的 network token 和稳定币的 token，根本不是一回事。
💳 卡组织 Network Token
◈ 稳定币 Token
本质真实卡号（PAN）的安全替身
本质一种链上资产 / 货币
跑在哪现有卡网络（Visa / MC 轨道）
跑在哪区块链
解决什么凭证安全 + 授权率 + 生命周期管理
解决什么价值转移 + 结算
对 Agent 的意义给 Agent 一个受约束、可撤销、可追溯的支付身份
对 Agent 的意义给 Agent 一笔可花的钱

agentic payment 真正缺的是前者——一个能安全交给 AI、还能随时收回的凭证。network token 恰好就是干这个的，而它顺带把成本也降了。下面是 token 化「降本增效」的四个具体环节：

① 授权率 · 最被低估的降本 提升授权成功率（authorization uplift）

每一笔 false decline 都是直接收入损失。network token 携带更新鲜的凭证信息（换卡时自动更新、不断链），发卡行批准率显著高于裸卡号。对 Agent 的含义：Agent 无人值守，被拒了没人在现场重输卡号——授权率就是 Agent 支付的生命线，而这是稳定币给不了的软成本优化。

② 安全 · 降欺诈与拒付 域限定的 token，爆炸半径趋近零

发给某 Agent / 某商户的 token，即使泄露也无法在别处盗刷。这把数据泄露损失、拒付处理成本、合规摩擦都压下去——而这些成本最终都摊在费率里。token 化让卡网络能在不牺牲安全的前提下，给可信交易更低的风险定价。

③ 持久性 · 消灭维护成本 凭证生命周期自动化（Account Updater）

传统卡号过期/挂失，绑定的自动扣费就断了。network token 自动续期——卡换了，token 不变，订阅与 Agent 授权不中断。对 Agent 的含义：授权一次就能长期稳定替你花钱，不会因换卡瘫痪。这种「免维护的持久委托」正是 agentic 经济的刚需，且零额外成本。

④ 接入 · 抹平易用性优势 Visa Intelligent Authorization（2026.3）

依托 Visa Acceptance Platform，收单机构「通过连接单一 API 就能接入智能化授权」，官方明确表述是「有效降低系统重建的成本并提升效率」。接入成本被压平——而接入成本恰恰是新轨道（含稳定币）最大的隐形门槛。

收口：卡网络不是一个等着被颠覆的静态高成本目标，而是一个正在自我优化、且优化方向（授权率、可撤销、免维护、低接入成本）恰好命中 Agent 痛点的活体系统。当 Agent 需要的「软指标」都被 token 化覆盖，稳定币那点「硬费率」优势在主流场景里就更撑不起「赢家必须用稳定币」的结论。
外部事实参考：Visa Intelligent Commerce；Visa Intelligent Authorization（2026-03 发布，单一 API 降低系统重建成本）；Stripe Shared Payment Tokens / ACP；Mastercard Agent Pay（tokenization + trust + agentic commerce）。授权率提升、域限定 token、Account Updater 为业内公认的 network tokenization 机制。

✅ 不用稳定币、仍做 agentic payment（本命题的证据）

  • Visa Intelligent Commerce、Mastercard Agent Pay——卡网络派，分发最广。
  • Stripe 虚拟卡 / Shared Payment Tokens——主流落地路径在卡轨道。
  • PayPal Agent Toolkit——复用既有账户/卡。
  • Google AP2、Amazon Bedrock AgentCore Payments——轨道中立 / 平台计费，本体不需要稳定币。

这些是当下分发覆盖最大、商户改造成本最低、对普通消费者零教育成本的方案。它们集体证明：做 agentic payment 完全可以不碰稳定币。

◈ 稳定币原生派（命题的反方 / 边界）

  • Coinbase x402、Stripe Tempo + MPP、Kite.AI、Skyfire、Circle。
  • 共同点：都瞄准 A2A / M2M 微支付——机器对机器、$0.001 级、高频、跨境、即时终局。
  • 这正是卡网络固定费与 T+N 结算的结构性短板。

诚实交底：如果「机器经济自循环」成为主体而非边缘，论据三、四的前提会崩塌——这也正是 Stripe 同时下注卡网络（虚拟卡）和稳定币（Tempo）两手的原因。

Verdict

结论：在主战场赢，在未来战场需对冲

Agentic payment 的瓶颈是「授权、信任、可追责、可回滚、双边覆盖」——五件事全是卡网络的主场，没有一件需要稳定币。卡组织在网络层发 Agent token，Stripe 发虚拟卡，主流场景下既够用、又有可逆性与合规优势，还自带数十亿用户与全球商户的网络效应。稳定币的「快、终局、便宜跨境」要么是 Agent 场景不需要的（终局性反而有害），要么是边缘市场（M2M 微支付），要么正被监管与卡组织自身降本抹平。

在「人替机器买现实世界商品」这个当下主战场，本命题几乎是赢的；
在「机器经济自循环（A2A 微支付）」这个未来战场会输。
谁是「最终赢家」，本质取决于你赌哪个场景成为主体——这正是巨头们卡网络与稳定币两手下注的原因。
